## Как запустить

### 1\. Собрать проект

```
Bash

go build -o cli ./cmd/cli
```

### 2\. Запустить

```
Bash

./cli https://github.com/gin-gonic/gin.git
```


### JSON-режим

Если нужен вывод в JSON:

```
Bash

./cli --json https://github.com/gin-gonic/gin.git
```



## Описание решения

1. Сначала утилита получает ссылку на Git-репозиторий и клонирует его во временную директорию.
Клонирование выполняется не в полном виде, а в облегчённом режиме:
- берётся только актуальное состояние репозитория без полной истории
- уменьшается объём загружаемых данных
- используется sparse-режим для более экономной работы

```go
cmd := exec.CommandContext(ctx, "git", "clone", "--depth=1", "--filter=blob:none", "--sparse", repoURL, dst)
```

Это позволяет быстрее подготовить репозиторий к анализу и не скачивать лишнее.


2. После клонирования программа открывает файл `go.mod`


3. Затем достаём имя модуля и версию Go


4.  Дальше программа запускает команду `go list -m -u -json all` внутри клонированного репозитория.
Это позволяет получить информацию по всем зависимостям проекта в формате JSON и понять, для каких из них есть более новая версия.
Команда запускается с GOWORK=off, чтобы анализировался именно текущий репозиторий, без влияния внешнего Go workspace.
Из полученного результата программа берёт только те зависимости, у которых поле обновления не пустое, и добавляет их в итоговый список


5. Затем формируем итоговый результат в 2 форматах:
- обычный текстовый
- JSON, если передан флаг --json
